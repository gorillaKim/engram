# 디자인 스펙: Desktop UI & Tray Popover Polish

## 1. 개요
Engram 데스크톱 앱과 트레이 팝오버(위젯)의 사용자 경험을 향상시키기 위한 UI 개선 작업입니다. 특히 트레이 팝오버에서 발생하는 스크롤 버그를 수정하고, 전반적인 시각적 완성도를 높이는 것을 목표로 합니다.

## 2. 주요 개선 사항

### 2.1 트레이 팝오버 (Tray Popover)
*   **스크롤 버그 수정**: 
    *   내용이 길어질 경우 헤더와 푸터는 고정되고, 중앙의 요약/알림/MCP 상태 영역만 스크롤되도록 구조를 변경합니다.
    *   하단 푸터("보드 열기...")가 스크롤 시에도 항상 하단에 고정되어 보더가 사라지는 문제를 해결합니다.
*   **시각적 정교화**:
    *   글래스모피즘(Backdrop blur) 효과를 정밀하게 조정하여 더 현대적인 느낌을 줍니다.
    *   최근 알림 항목에 호버 효과를 추가하여 인터랙티브한 피드백을 제공합니다.
    *   타이포그래피 위계와 여백을 조정하여 가독성을 높입니다.

### 2.2 메인 데스크톱 창 (Main Window)
*   **헤더 내비게이션 개선**:
    *   현재의 단순한 버튼 형태에서 더 세련된 Segmented Control 또는 사이드바 형태로 전환을 검토합니다.
    *   내비게이션 요소의 대비와 활성 상태 표시를 명확하게 합니다.
*   **칸반 보드 비주얼 개선**:
    *   이슈 카드의 그림자(Shadow), 테두리 곡률(Border-radius), 내부 여백을 조정하여 깔끔한 룩을 구현합니다.
    *   컬럼 간의 구분과 빈 상태(Empty State) 디자인을 개선합니다.

### 2.3 일관성 (Consistency)
*   트레이 위젯과 메인 창에서 사용하는 포인트 컬러(Indigo, Amber 등)와 디자인 시스템 요소를 통일합니다.

## 3. 기술적 구현 계획
*   **Tray Scroll**: `max-h`와 `overflow-y-auto`를 적용한 내부 컨테이너 도입.
*   **Tailwind CSS**: 기존 테일윈드 설정을 활용하여 정교한 유틸리티 클래스 적용.
*   **Framer Motion**: (선택 사항) 트레이 팝업 오픈 및 알림 리스트 업데이트 시 부드러운 애니메이션 추가.

## 4. 수락 기준 (Acceptance Criteria)
*   [ ] 트레이 팝오버에서 알림이 많아져도 스크롤이 정상적으로 작동하며 푸터가 고정됨.
*   [ ] 트레이 팝오버의 푸터 상단 보더가 스크롤 시에도 유지됨.
*   [ ] 메인 창 헤더가 더 현대적인 디자인으로 변경됨.
*   [ ] 전반적인 UI 요소(그림자, 여백, 폰트)가 일관성 있게 적용됨.
